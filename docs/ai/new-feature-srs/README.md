# New-Feature SRS

Drop the spec for the **feature under test** here as Markdown/text.

Guidelines:
- Give each use case a heading and ID, e.g. `## UC-N01 — Save a payment method`.
- State acceptance criteria for each use case.
- You do NOT need to author relationships to the existing SRS — the
  **generate-test-cases** skill discovers related existing use cases, confirms
  them with you, and writes the pointers into `_index.md`.

This folder ships empty. Provide the real new-feature spec.
