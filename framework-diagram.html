How to read it

  • Top diagram = the pipeline. A single chosen use case flows top-to-bottom: index once → generate → execute in a real browser → triage failures → review gate → upload to Azure DevOps → promote back into the baseline SRS (left dashed feedback line).
  • Right column = what persists between sessions. Downstream stages read these, not the SRS.
  • Amber path is gated: nothing reaches Azure DevOps until the review gate writes review-passed.json.
  • Bottom diagram = enforcement. Five hooks run outside the model and can DENY an action — they turn "please follow the rules" into rules that hold.
  • Token efficiency: the full SRS is read only at stage 1 (once). Every test run reads tiny index files plus one use-case section and its confirmed related sections.
